### 2.2 F2 — 报告生成器

**功能摘要**：触发引擎按规定顺序执行全部 5 个 MRC 验证器，并将输出渲染为与旧系统 XLSX 单元格等价的工作簿；F3–F8 功能在成功后解锁。

**Stage 1 锚点**：ch 1.2 § 2（验证器执行顺序）· ch 1.3 § 3（渲染管道）· ch 1.6（baseline 契约）

---

#### FR-F2.1 — 验证器执行顺序

| 属性 | 内容 |
|---|---|
| **描述** | 引擎按以下固定顺序依次执行 5 个 MRC 验证器：`mrc_summary_check` → `mrc_check_general_info` → `mrc_check_adv_balance` → `mrc_service_fee_check` → `mrc_other_check`。 |
| **输入** | G2a 冻结的本地输入快照（`baselines/mrc/<remit_date>/input_snapshots/`）；F1 四个参数 |
| **输出** | 5 个验证器各自的 DataFrame 结果 |
| **前置条件** | G2a 已关闭；F1 四参数已填充；引擎代码可访问本地快照 |
| **后置条件** | 5 个 DataFrame 结果全部产出；可传递给渲染管道（FR-F2.2） |
| **错误处理** | 若某验证器抛出异常，停止执行剩余验证器，记录异常到运行日志，向 UI 报告"第 X 步失败"并展示错误摘要 |
| **B1 追溯** | B1 F2 验收标准第 1 条 `[CONFIRMED]`（`remit_validation.py:134-144`） |
