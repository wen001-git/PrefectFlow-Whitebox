# Stage 2 Design Package — Overview

## 1. What Is Stage 2

Stage 2 builds on the Stage 1 MRC reverse-engineering documentation to **design and implement from scratch** a new MRC Validation Report generation system that satisfies:

1. **Cell-identical output**: for a fixed `remit_date` baseline, the new system's XLSX matches the legacy system byte-for-byte on sheet names, sheet order, column order, header rows, every cell value, and highlight colors / conditional formatting. Verified by `stage2-mrc-cell-identity-harness`.
2. **8-feature interactive UI**: an interactive interface covering the 8 features specified in prompt 19 (see `1.0-feature-list.en.md`).
3. **Extensible to other servicers**: the engine, data model, UI, and registry all reserve slots for Arvest / CC5 / Selene / SLS / etc. so future onboarding is incremental, not a rewrite.

## 3. Hard-Gate Gating Conditions

Implementation work (writing code, running harness tests) **must not start** until all three gates below are closed:

| Gate | Definition | Status | Owner |
|---|---|---|---|
| **G1** User walk-through 1.0–1.6 | User sign-off on all Stage 1 MRC chapters | ✅ **Closed** (provisional sign-off + AGENTS.md § 6.11 living-docs rule) | User |
| **G2a** Local input-snapshot freeze | Export all Redshift queries/tables the MRC validators read for `2026-04-30` to local Parquet under `baselines/mrc/2026-04-30/input_snapshots/` | ⏸ **Pending** (user action — requires Redshift access) | User |
| **G2b** Reproduce baseline XLSX from legacy code | Build Parquet shim; run unmodified legacy MRC code against frozen snapshot; produce XLSX; cross-check vs LearningLog gold | ⏸ **Blocked on G2a** | Agent |
| **G3** V1–V12 assertions upgraded to `[CONFIRMED]` | Physically verify Stage 1 `[VERIFY]`-tagged assertions against the G2b baseline XLSX | ⏸ **Blocked on G2b** | Agent |

> **Design layer (B1–B6 documents) can proceed immediately** — the gate restriction applies only to implementation code.

## 5. Relationship to Stage 1

Every document in this package treats the Stage 1 chapters under `docs/mrc/` as **read-only references** (cite as `see ch 1.N § M`). This package does not modify `docs/mrc/` or `tools/`. If Stage 2 investigation reveals a Stage 1 inaccuracy, it is corrected per AGENTS.md § 6.11 in a separate commit, independent of this design package.

## 6. Servicer Extension Placeholders

Everywhere in this package that refers to "supported servicers", the AGENTS.md § 6.8 rule applies:

- **MRC**: the only currently implemented servicer.
- **Arvest / CC5 / Selene / SLS / Scattered / Cross-servicer dataflow**: all `⏳ pending-analysis`; explicit placeholders are reserved in B4 UI design, B5 dev plan, and B6 extensibility spec.
