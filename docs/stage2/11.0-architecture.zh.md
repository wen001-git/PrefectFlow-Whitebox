# Stage 2 — 架构冻结（P2.0）

**状态**：自 2026-05-18 起 FROZEN。
**依据**：plan.md § 10、decisions.md 中"Round 3 architecture freeze"条目。
**变更政策**：对本文档的任何修改都需要在 `decisions.md` 新增 ADR + 对应修订 plan.md。

## 1. 顶层布局

```
PrefectFlow-Whitebox/
├── whitebox/                  # Python 源码根（已存在，扩展）
│   ├── validators/            # V1..V12 校验器实现（已存在）
│   ├── registry/              # 新增 — 校验器/服务商/sheet/数据集注册表
│   ├── ingestion/             # 新增 — Redshift 适配器 + 冻结快照 loader + CTE 重放
│   ├── transform/             # 新增 — 纯 DataFrame 变换（无 IO）
│   ├── engine/                # 新增 — 编排 registry + transform + validators
│   ├── sheets/                # 新增 — 每张 sheet 的 builder（5 张 MRC，可扩展）
│   ├── renderer/              # 新增 — openpyxl XLSX 写出（cell-identical）
│   └── api/                   # 新增 — FastAPI 应用（见 § 4）
├── apps/
│   └── web/                   # 新增 — Next.js App Router 前端（见 § 5）
├── tools/                     # CLI 工具（已存在）
├── tests/                     # pytest 套件（已存在）
├── docs/stage2/               # Stage 2 设计文档（已存在）
├── baselines/                 # 冻结基线输出 + 快照（已存在）
├── runs/                      # 每次运行的产物（gitignore）
└── .github/                   # CI + PR 模板（新增）
```

**理由**：现有 Python 包保持原样，新增子包都放进 `whitebox/`，Next.js 单独落在 `apps/web/`。不移动任何已存在文件。

## 2. 版本钉死（HARD）

| 工具 | Pin | 理由 |
|---|---|---|
| Python | `>=3.10,<3.11` | 已钉死；Prefect 生态兼容 |
| openpyxl | `==3.1.5` | **Cell-identity 稳定性**。Round 2 C5 harness 验证过。仅允许通过 ADR + 完整 cell-identity 复验 后变更。 |
| FastAPI | `>=0.115` | 稳定的异步 API surface |
| Node | `>=20.0,<21.0` | Next.js 14+ LTS 要求 |
| Next.js | `^14.2` | App Router 已稳定 |

## 3. 模块边界

| 模块 | 拥有 | 不做 |
|---|---|---|
| `ingestion` | 数据源连接、快照 loader、CTE 重放 | 校验逻辑、格式化 |
| `transform` | 按 `3.0-data-model` 做纯 DataFrame 整形 | IO、副作用 |
| `registry` | 校验器/服务商/sheet/数据集 注册与查找 | 执行 |
| `engine` | 编排：按 servicer discriminator 调度校验器 | 直连 DB、渲染 |
| `sheets` | 由校验器输出构建预渲染 sheet model | XLSX 字节流 |
| `renderer` | openpyxl writer；cell-identical 保真 | 业务决策 |
| `api` | 给 FE 用的 HTTP 表面；engine 的薄层 | 业务逻辑 |
| `apps/web` | UI；消费 API | 校验规则、数据整形 |

## 4. 后端（FastAPI）

**布局**：`whitebox/api/`，路由按 runs/sheets/diff/lineage/export 拆分。
OpenAPI 在 `/openapi.json` 公开，FE 用它做类型生成。

## 5. 前端（Next.js）

**布局**：`apps/web/`，App Router 默认 server components。
- 起步只引入：App Router + Tailwind + 一个图库（react-flow 或 cytoscape）
- **克制原则**：开头不要 Redux/Zustand/tRPC，等真痛了再加，加之前先 ADR

## 6. PR 证据规则（HARD）

任何改动 `whitebox/engine|sheets|renderer|validators|transform|ingestion|registry`
的实现 PR，**必须**在 PR 描述里包含：

1. 实际跑的 `python tools/compare_validation.py auto ...` 命令
2. 产出的 `verdict.json` 内容（verdict: PASS / MINOR_DIFFS / MAJOR_DIFFS / ERROR）
3. 若 MINOR_DIFFS：必须显式说明 + ADR 引用
4. 若 MAJOR_DIFFS：PR 不得 merge

通过 `.github/PULL_REQUEST_TEMPLATE.md` checklist + reviewer 责任执行。

## 7. P2.0 范围外（延后）

- 多租户：硬假设单租户
- 审计日志：schema 预留，实现延后
- 实时 push（websocket）：等真有需要再做
