## 1. 接受的范围（Scope acknowledged）

Stage 2 v9.1 = 面向 MRC 的校验系统，交付内容：

1. prompt-19 的 8 个交互式 UI 功能，限定在 MRC 范围内。
2. `validation_report.xlsx` 与遗留 PrefectFlow MRC 报表**逐单元格一致**
   （cell-identical），由 `tools/acceptance_gate.py` 的 self-consistency
   底线 + baseline-diff 层（当有已捕获 XLSX 时）保证。
3. 架构层面的可扩展性 — registries、servicer discriminator、sheet/
   validator 插件以及待分析 servicer 占位 — 让 Carrington / Arvest /
   CC5 / Selene / SLS / Shellpoint 后续按 servicer 逐个 onboard，无需重写。

**本签字仅覆盖 MRC**。其他 servicer 仍位于
`docs/_status/servicers-registry.{zh,en}.md` 与 validator 注册表的
`pending-analysis` 槽位中；它们的 onboarding 将以各自的签字文档落地。

## 9. 签字声明（Sign-off statement）

**MRC Stage 2 v9.1 接受为开发基线（development baseline）**，前提：

a. § 2 中每条标准都为 PASS（或像 ruff 那样明确标注为 out-of-scope
   legacy 代码的 MARGINAL）。
b. 操作员在任何生产切换前，先捕获真实 baseline XLSX 并以
   `--baseline` 模式（最终以 `--legacy-mode live`）重跑 acceptance
   gate。
c. DEGRADED validator 增量补齐；每个替换 PR 必须按 AGENTS.md § 6.13
   带 `tools/compare_validation.py` 证据。
d. 每个真实 validator 实现都以单独 PR 落地，并附 acceptance-gate 证据。
