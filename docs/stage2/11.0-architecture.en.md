# Stage 2 — Architecture Freeze (P2.0)

**Status**: FROZEN as of 2026-05-18.
**Authority**: plan.md § 10, decisions.md "Round 3 architecture freeze" entry.
**Change policy**: any modification to this document requires a new ADR in `decisions.md` and a corresponding plan.md revision.

---

## 1. Top-level layout

```
PrefectFlow-Whitebox/
├── whitebox/                  # Python source root (existing, extended)
│   ├── validators/            # V1..V12 validator implementations (existing)
│   ├── registry/              # NEW — validator/servicer/sheet/dataset registries
│   ├── ingestion/             # NEW — Redshift adapters + frozen-snapshot loaders + CTE replay
│   ├── transform/             # NEW — pure DataFrame transforms (no IO)
│   ├── engine/                # NEW — orchestrates registry + transform + validators
│   ├── sheets/                # NEW — per-sheet builders (5 MRC sheets, extensible)
│   ├── renderer/              # NEW — openpyxl-based XLSX writer (cell-identical)
│   └── api/                   # NEW — FastAPI app (see § 4)
├── apps/
│   └── web/                   # NEW — Next.js App Router frontend (see § 5)
├── tools/                     # CLI utilities (existing): freeze_snapshot, xlsx_diff,
│                              # compare_validation, run_legacy_mrc, run_newsystem_mrc
├── tests/                     # pytest suite (existing)
├── docs/stage2/               # Stage 2 design docs (existing)
├── baselines/                 # Frozen baseline outputs + snapshots (existing)
├── runs/                      # Per-run artifacts (gitignored)
└── .github/                   # CI + PR templates (NEW)
```

**Rationale**: keep the existing Python package intact, add new sub-packages
inside `whitebox/`, add the Next.js island under `apps/web/`. No file moves.

## 2. Pinned versions (HARD)

| Tool | Pin | Reason |
|---|---|---|
| Python | `>=3.10,<3.11` | Already pinned; Prefect ecosystem compatibility |
| openpyxl | `==3.1.5` | **Cell-identity stability**. Proven by Round 2 C5 harness. Bump only via ADR + full cell-identity re-verification. |
| FastAPI | `>=0.115` | Stable async API surface |
| Node | `>=20.0,<21.0` | Next.js 14+ LTS requirement |
| Next.js | `^14.2` | App Router stable |

## 3. Module boundaries

| Module | Owns | Does NOT do |
|---|---|---|
| `ingestion` | Source connectivity, snapshot loaders, CTE replay | Validation logic, formatting |
| `transform` | Pure DataFrame shaping per `3.0-data-model` | IO, side effects |
| `registry` | Validator/servicer/sheet/dataset registration + lookup | Execution |
| `engine` | Orchestration: dispatch validators per servicer discriminator | Direct DB calls, rendering |
| `sheets` | Build pre-render sheet model from validator outputs | XLSX bytes |
| `renderer` | openpyxl writer; cell-identical fidelity | Logic decisions |
| `api` | HTTP surface for FE; thin layer over engine | Business logic |
| `apps/web` | UI only; consumes API | Validation rules, data shaping |

## 4. Backend (FastAPI)

**Layout**: `whitebox/api/`
- `main.py` — app + router registration
- `routers/runs.py` — list_runs / get_run
- `routers/sheets.py` — get_sheet / get_cell
- `routers/diff.py` — diff between two runs (wraps `tools/xlsx_diff.py`)
- `routers/lineage.py` — field lineage graph (from registry metadata)
- `routers/export.py` — download XLSX
- `deps.py` — DI shims (auth, db session)

**OpenAPI** published at `/openapi.json` for FE codegen.

## 5. Frontend (Next.js)

**Layout**: `apps/web/`
- `app/` — App Router pages (server components by default)
- `app/page.tsx` — picker (remit_date + servicer)
- `app/runs/[runId]/page.tsx` — run results browse
- `app/runs/[runId]/sheets/[sheetName]/page.tsx` — per-sheet drill-down + pass/fail panel
- `app/runs/[runId]/lineage/page.tsx` — lineage graph (react-flow)
- `app/runs/[runId]/diff/page.tsx` — XLSX diff viewer
- `lib/api.ts` — fetch wrapper, typed against OpenAPI
- `components/` — small set; NO Redux/Zustand/tRPC at start (per plan § 10.2)

**Restraint**: minimum library set. Add complexity only when a real need arises (documented in ADR).

## 6. PR evidence rule (HARD)

Every implementation PR that touches `whitebox/engine`, `whitebox/sheets`,
`whitebox/renderer`, `whitebox/validators`, `whitebox/transform`,
`whitebox/ingestion`, or `whitebox/registry` **MUST** include in PR description:

1. The exact `python tools/compare_validation.py auto ...` invocation used
2. Resulting `verdict.json` content (verdict: PASS / MINOR_DIFFS / MAJOR_DIFFS / ERROR)
3. If MINOR_DIFFS: explicit justification + ADR reference
4. If MAJOR_DIFFS: PR must NOT be merged

Enforced via `.github/PULL_REQUEST_TEMPLATE.md` checklist + reviewer responsibility.

## 7. Deferred / out-of-scope (P2.0)

- Auth: stub only; full SSO deferred to post-MVP
- Multi-tenant: single-tenant assumption baked in
- Audit log: schema reserved, implementation deferred
- Realtime updates (websockets): defer until proven needed
