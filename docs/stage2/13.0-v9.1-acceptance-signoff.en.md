## 1. Scope acknowledged

Stage 2 v9.1 = an MRC validation system delivering:

1. The 8 prompt-19 interactive UI features, scoped to MRC.
2. Cell-identical XLSX output (`validation_report.xlsx`) vs the legacy
   PrefectFlow MRC report, enforced by `tools/acceptance_gate.py`
   self-consistency floor + baseline-diff tier (when a captured XLSX
   is present).
3. Architectural extensibility — registries, discriminators, sheet/
   validator plugins and pending-servicer placeholders — so onboarding
   Carrington / Arvest / CC5 / Selene / SLS / Shellpoint becomes
   incremental, not a rewrite.

**This sign-off covers MRC only.** Every other servicer remains in the
`pending-analysis` slot of `docs/_status/servicers-registry.{zh,en}.md`
and the validator registry; their onboarding will land via their own
sign-off documents.

## 7. Known gaps / deferred items

- **a) G2a (Redshift snapshot freeze) — deferred, environment-blocked.**
  Operator runbook stays alive at `tools/docs/g2a-operator-runbook.{en,zh}.md`.
  Tooling (A1–A6) is complete; only the credentialled export remains.
- **b) Real baseline XLSX not yet captured.** `baselines/mrc/2026-04-30/CAPTURE_INSTRUCTIONS.md`
  awaits an operator with VPN + Vault token. Until then, the gate's
  baseline-diff tier is ENV-SKIP (documented, not a failure).
- **c) Validators V1–V12 ship as stubs / DEGRADED on the bundled CTE
  harness.** `mrc_service_fee_check` runs clean; the other 4 mark
  `DEGRADED` because fixture schemas are partial (binder/catalog
  mismatch). Real validator logic awaits real data.
- **d) Lineage payload is fixture-backed.** Engine-side lineage emission
  is deferred to v9.2; FE lineage screens consume `whitebox/api/data/fixtures.py`.
- **e) Export endpoint returns 501.** Engine-side XLSX bytes streaming
  is deferred; CLI `python -m whitebox.engine` is the supported export
  surface for v9.1.
- **f) `12.0-acceptance-gate.zh.md` missing.** All other Stage 2 docs
  ship as zh+en pairs; 12.0 is currently English-only.
- **g) 34 pre-existing ruff errors in legacy/Round-2 tooling files.**
  Outside the Round 3 scope but tracked here so they are not forgotten.

None of (a)–(g) blocks the v9.1 acceptance under the criteria of § 2,
but each is a concrete follow-up (see § 8).
